# WiFi Термостат + ОПС

Проект умного термостата и охранно-пожарной сигнализации на базе ESP32 (ESP-IDF).

## Возможности

### Термостат
- Управление котлом отопления по нескольким режимам:
  - **Ручное включение/выключение** — котел работает постоянно
  - **По расписанию** — управление по времени суток
  - **По температуре** — включение/выключение при достижении порога
  - **Расписание + температура** — комбинированный режим
  - **Интерполяция** — автоматический расчёт температуры котла по графику в зависимости от уличной температуры
- Настраиваемый гистерезис
- MQTT управление и мониторинг

### Датчики

| Датчик | Параметры | Интерфейс |
|--------|-----------|-----------|
| Улица | AM2302 (DHT22) — температура, влажность | GPIO |
| Комната | BME280 — температура, влажность, давление | I2C |
| Котёл | DS18B20 — температура теплоносителя | OneWire |

### Мониторинг
- MQTT брокер — основной канал управления и мониторинга
- OpenMonitoring.online — отправка телеметрии
- NarodMon.ru — отправка телеметрии
- ThingSpeak — отправка телеметрии

### Уведомления
- Telegram бот — оповещения о выходе температуры за пределы диапазона, изменении состояния котла и событиях охранной сигнализации

## Режимы работы термостата

### Интерполяция (рекомендуемый)

Температура котла рассчитывается автоматически по графику:

```
Температура котла = f(температура на улице)
```

По умолчанию используется 6 точек:

| Точка | Улица (°C) | Котёл (°C) |
|-------|------------|------------|
| P1    | -30        | 70         |
| P2    | -20        | 65         |
| P3    | -10        | 58         |
| P4    | 0          | 42         |
| P5    | 10         | 35         |
| P6    | 20         | 25         |

Линейная интерполяция между точками обеспечивает плавное изменение температуры котла в зависимости от погоды.

### MQTT топики

#### Управление термостатом

| Параметр | Топик | Описание |
|----------|-------|----------|
| Режим | `thermostat/mode` | 0=Выкл, 1=Вкл, 2=Расписание, 3=Температура, 4=Расписание+Температура, 5=Интерполяция |
| Уставка | `thermostat/temperature` | Целевая температура (°C) |
| Гистерезис | `thermostat/hysteresis` | Гистерезис (°C) |
| Расписание | `thermostat/timespan` | Временной диапазон работы |
| Уведомления | `thermostat/notifications` | Включить/выключить уведомления |
| Состояние котла | `thermostat/boiler` | Текущее состояние (on/off) |

#### Настройка точек интерполяции

| Параметр | Топик |
|----------|-------|
| Улица P1 | `thermostat/interpolation/p1_out` |
| Котёл P1 | `thermostat/interpolation/p1_bo` |
| ... | ... |
| Улица P6 | `thermostat/interpolation/p6_out` |
| Котёл P6 | `thermostat/interpolation/p6_bo` |

#### Мониторинг датчиков

| Датчик | Топик |
|--------|-------|
| Улица (темп) | `outdoor/temp` |
| Улица (влаж) | `outdoor/humidity` |
| Комната (темп) | `indoor/temp` |
| Комната (влаж) | `indoor/humidity` |
| Комната (давл) | `indoor/pressure` |
| Котёл | `boiler/temp` |

## Сборка

### Требования

- PlatformIO Core или VS Code с PlatformIO IDE
- ESP-IDF 5.x
- Библиотеки (см. `libs_local_*.rar`)

### Компиляция

```bash
pio run
```

### Прошивка

```bash
pio run --target upload
```

## Конфигурация

Основные настройки в `include/project_config.h`:

- `CONFIG_MQTT_*` — подключение к MQTT брокеру
- `CONFIG_WIFI_*` — параметры WiFi
- `CONFIG_TELEGRAM_*` — токен Telegram бота
- `CONFIG_GPIO_*` — назначение GPIO пинов
- `CONFIG_SENSORS_TASK_*` — период опроса датчиков

## Как начать работу

1. Создайте на диске C каталог `PlatformIO` → `C:\PlatformIO\`
2. Скачайте нужную ветку репозитория в виде ZIP и распакуйте в `C:\PlatformIO\telemeter_dzen`
3. Распакуйте `libs_local_*.rar` в `C:\PlatformIO\libs`
4. Откройте проект в VS Code и скомпилируйте
